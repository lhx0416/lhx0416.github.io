1.TinyWebServer

传送门：GitHub - qinguoyi/TinyWebServer: Linux下C++轻量级Web服务器、牛客视频

  • GCC/G++ 编译 （预处理 编译 汇编 链接）

  • 静态库的制作与使用 ：gcc/g++进行链接时，会把静态库中代码打包到可执行程序中（先得到.o文件，然后利用ar rcs libxxxx.a xxx.o 制作一个库文件，只需要保留.h头文件和.a库文件，别人就可以使用程序，同时实现了对源码的保护，别人只能用，但是看不到源码）

  • 动态库的制作与使用 ：gcc/g++进行链接时，不会把动态库中代码打包到可执行程序中，而是放了一些信息，比如名称之类的。当程序运行起来的时候用到动态库里的API时，会启动动态载入器获取动态库的绝对路径，之后加载动态库到内存中使用（先得到.o文件，然后利用g++ -shared xxx.o -o libxxx.so制作一个动态库文件）

    静态库的优缺点：

    • 直接打包到应用程序中，加载速度快
    • 发布应用程序无需提供静态库，方便移植
    • 消耗系统资源，浪费内存
    • 更新、部署、发布麻烦

    动态库的优缺点：

    • 可以实现进程间资源共享（共享库）
    • 更新、部署、发布简单（自己的程序不需要重新编译一次，只需要更新库文件就好了）
    • 可以控制何时加载动态库
    • 加载速度比静态库稍慢一点
    • 发布程序时需要提供依赖的动态库

2.CPlusPlusThings

这是一个适合 C++ 初学者从入门到进阶的教程。

解决了面试者想要深入 C++ 及如何入坑 C++ 的问题。

除此之外，该仓库拓展了更加深入的语法分析、多线程并发等的知识，是一个比较全面的 C++ 从入门学习到进阶提升的项目。
